Run -> Install -> Verify

ONBOARDING

Treat the first healthy loop like a three-step lane: build one baseline, install the schedule, then verify before you read deeper telemetry.

Step 1 Run one snapshot

Use Run Snapshot first so macOS can show permissions and the local ledger can record a real baseline.

Step 2 Install the scheduler

Use the Scheduler card once the first run succeeds so the loop becomes repeatable instead of manual.

Step 3 Verify before diagnostics

Verify first, then use Doctor, Recent Runs, and Log Health only when the loop still looks wrong.

Need the guided version? Open the quickstart, then the troubleshooting guide, and keep the proof page for after the first verified loop.

First-Run Guidance

--

Build one successful snapshot first, then treat the rest of the control room as ongoing operations.

  1. Run ./notesctl run --no-status so macOS can surface the first permission prompts.
  2. Run ./notesctl install --minutes 30 --load once the first snapshot succeeds so the loop becomes repeatable.
  3. Run ./notesctl verify to confirm the local state ledger and scheduler now agree on a healthy baseline.
Need the full checklist? Open the quickstart or the troubleshooting guide.
