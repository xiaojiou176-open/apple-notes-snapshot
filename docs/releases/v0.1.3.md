# Apple Notes Snapshot v0.1.3

## What changed after v0.1.2

This release tightens the support-routing layer so users land in the right
place faster instead of opening the wrong kind of issue.

## Highlights

- a root `SUPPORT.md` file for support routing in the repository itself
- a public support page on the docs site
- stricter issue routing through contact links and disabled blank issues
- better guidance for setup questions, launchd questions, and reproducible bugs

## Why it matters

Once a project becomes easier to discover, the next challenge is helping people
ask in the right place. `v0.1.3` makes the support surface more deliberate so:

- setup help flows to Q&A
- roadmap ideas flow to Ideas
- real examples flow to Show and tell
- reproducible repo bugs still get issue templates
