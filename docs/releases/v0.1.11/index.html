Side-room routing polish

v0.1.11 makes the side rooms feel like the same exhibition as the front door.

This release does not widen product scope. It tightens the public routing and release truth so Proof, Security, Support, Community, and Troubleshooting all read with the same control-room-first rhythm as the rest of the site.

Highlights

  • Polished side-room routing across Proof, Security, Support, Community, and Troubleshooting.
  • Made support-desk, discussion-hall, and runbook pages feel more like one coherent public surface.
  • Closed the release-truth gap so the latest public tag, proof page, release history, and changelog now agree again.

Why it matters

The front door was already strong after `v0.1.10`. What still felt unfinished was the transition into the side rooms: support, community, troubleshooting, and proof-adjacent pages did not always feel like they belonged to the same carefully curated exhibition. `v0.1.11` fixes that without changing what the repo is.
