Keep the layers separate

The latest public tag is currently v0.1.12. At the current baseline, the latest public tag and current main can move at different speeds. Keep treating CHANGELOG Unreleased as the moving main-branch preview after the next tag is cut, and do not collapse local branch freshness into tagged release truth.

Read the latest tag as the stable public exhibit. Read current main as the workshop next door. When you need evidence behind a release claim, jump to Proof instead of turning the release table into a substitute for verification.

If you are preparing a public update from current main

  • Proof for the repo-side gates, GitHub-controlled evidence, and live-boundary summary
  • Share kit for the punchline, demo clip, and compare card
  • 30-second overview for the shortest accurate summary
  • External launch prep for the repo-ready checklist versus manual-only actions
  • Codex starter pack, Claude Code starter pack, and Public skills pack for the builder-side tagged truth
  • Community for the announcements lane and discussion routing
  • Support so release notes and announcements can still point people to the right help surface

Treat this as a repo-side preparation stack. It does not mean Search Console, domain changes, video uploads, or platform distribution are already complete.

Use release history for tagged truth, the share kit for reusable assets, and the external launch prep packet when you are one step away from manual launch actions.
