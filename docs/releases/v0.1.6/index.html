Latest release

v0.1.6 moves AI Diagnose onto a local Switchyard runtime.

This release keeps Apple Notes Snapshot as the local control room while handing model invocation off to a local Switchyard runtime and tightening the builder-proof wording.

Highlights

  • ./notesctl ai-diagnose now routes model calls through a local Switchyard runtime.
  • Deterministic fallback still works when the Switchyard runtime is unavailable or misconfigured.
  • AI Diagnose docs/help/config now explain the local Switchyard boundary and current Gemini BYOK path.
  • The Builder Integration Pack now treats Codex and Claude Code as generic-path documented hosts, not attach-proven hosts.

Why it matters

Apple Notes Snapshot still owns the backup control room. This tag only changes where the optional AI power comes from. The repo no longer carries direct OpenAI/Ollama transport details in its own AI Diagnose path.
