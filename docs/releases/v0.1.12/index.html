Builder closeout turn

v0.1.12 turns the builder-side closeout work into tagged truth.

This release keeps the control-room product identity intact while making the builder side feel more finished: Codex and OpenClaw now have fresh current-host attach proof, the public skills lane has a standalone control-room packet, and the official MCP Registry lane now has a real package path instead of just a plan.

Highlights

  • Fresh current-host attach proof for both Codex and OpenClaw.
  • A standalone public skill packet for notes-snapshot-control-room.
  • A tracked MCP Registry lane with server.json, an MCPB manifest, a build script, and a locally verified artifact path.

Why it matters

Before this tag, the repo had the builder wiring kit but not the same level of tagged closure around named-host proof and registry packaging. v0.1.12 closes that gap without pretending every machine, registry, or listing surface is already done.
