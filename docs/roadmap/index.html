Current direction

The project is getting better at operations calm, not pretending to become a cloud product.

This roadmap is intentionally narrow. It tracks the next useful layers around a local, macOS-bound Apple Notes snapshot workflow instead of promising a whole different product.

Now

The public launch surface, docs site, releases, discussions, release history, support routing, examples, troubleshooting, the path-first quickstart, a token-gated Local Web API, an opt-in AI Diagnose layer, and a read-only-first MCP provider are in place.

Next

Sharper screenshots, richer launchd / Web-console guidance, more recent-run summaries in the Web console, and carefully scoped MCP polish as real users arrive.

Later

More operational polish only if it stays reviewable, host-local, and honest about scope.

Likely near-term improvements

  • More example workflows for common Apple Notes export habits after the first successful run
  • More troubleshooting runbooks for launchd edge cases, permission prompts, and path surprises
  • Clearer AI Diagnose guidance and summary surfaces around recent runs, failure reasons, and operator next steps
  • Sharper integration-facing docs around the CLI substrate, Local Web API, MCP surface, and where agent integrations truthfully begin today
  • Better screenshots and short clips when the runtime surface grows in meaningful ways
  • Issue and discussion triage refinements based on real user traffic
  • Read-first MCP polish, plus any future guarded write-actions only when the current read-only surface proves useful

The first two items now have dedicated public entry points: example workflows and troubleshooting.

Hard boundaries that still hold

  • No cloud sync product positioning
  • No promise of two-way sync back into Apple Notes
  • No fake benchmark marketing
  • No papering over local security or path assumptions with vague copy
